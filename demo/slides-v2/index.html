从提需求到自己造

核心变化：做工具的成本崩塌了

以前
  • 有想法 → 提需求 → 排期 → 开发 → 上线
  • 周期：几周到几个月
  • 成本高：开发资源稀缺
  • 改不动：改一版又是一轮
现在
  • 有想法 → 描述问题 → AI 写 → 迭代 → 上手用
  • 周期：几十分钟到几小时
  • 成本低：你自己就是开发
  • 随时改：改 prompt，重跑

从 「我需要什么功能」 → 「我要解决什么问题」

方法论：先聊 Plan → 几轮讨论细化需求 → 敲定方案 → AI 实施 → 你来验收

方法论：搭 Agent 的四步

1

定角色

人设 + 能力边界
它擅长什么、不擅长什么

2

注入知识

领域知识 + 技能模块
但别一次塞太多（教训）

3

接工具

外部数据 + 内部平台
让它能查、能算、能存

4

持续进化

Reflection + Memory
用得越多，越懂你的场景

踩过的坑

Token 有限 — 别当硕士培养，用 RAG 动态匹配 任务拆细 — 越复杂越容易翻车，一步一步来 幻觉问题 — 做了 Claim Verification 核实关键论点

总结

三条方法论

一

工具箱思维

先解决眼前问题
再沉淀成通用工具
别一上来就想做平台

二

从具体到通用

先做透一个场景
再拆模块、抽框架
通用性是长出来的，不是设计出来的

三

让 AI 有记忆

知识 + 反思 + 记忆
让它越用越懂你的场景
这才是长期价值
