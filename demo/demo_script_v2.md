## 🎬 开场：我是谁，我在干什么（3 min）

大家好，我是 [名字]，在腾讯做外汇相关的工作。

简单说一下我平时干嘛的 —— 外汇交易这块涉及的东西挺杂的：定价、损益分析、策略研究、市场研判、数据平台运维……每一样都有大量重复性的分析和操作。

大概一年前，我开始琢磨一件事：**这些活儿里面，有多少是我非得亲自干的，又有多少是可以让 AI 帮忙的？**

今天不讲高大上的理论，就聊聊我自己的实践 —— 我把日常工作分成了几大类，然后一块一块地试着用 AI 来提效。有成功的，有翻车的，都跟大家分享一下，希望不管你是什么岗位，都能找到一些可以借鉴的方法。

| 📊 | **分析提效** | 日常重复性的数据分析，用 AI 做工具箱 |

## 📊 板块一：分析提效 — 工具箱思维（10 min）

### 核心方法论

先说一个我觉得最重要的心得：

> **不要一上来就想着做一个"平台"。先解决眼前的问题，跑通了，再让 AI 帮你把它变成一个通用的工具。**

我管这叫"工具箱思维" —— 遇到一个重复性的活儿，先针对这个具体问题写一个最快的解法。跑通了，再回头看：这类需求有通用性吗？如果有，就让 AI 帮我把它总结成一个可复用的 Skill。

下次再遇到类似的事，直接调这个 Skill 就行了。

### 案例一：PnL Analysis — 月报分析从 2 小时到 2 分钟（深讲）

**以前怎么干的？**

每个月月报，要把损益明细的 CSV 拿过来，按渠道分一下、按产品分一下、按币种分一下、按商户类型分一下……9 个维度，每个都要在 Excel 里折腾。折腾完还要做成汇报用的格式，加表头、加合计行、调颜色。

一套下来，快的话 2 小时，慢的话半天。而且每个月都来一遍。

**现在怎么干的？**

我做了一个 Web 工具 —— 把 CSV 拖进去，选好时段，点一下，9 个维度的汇总分析直接出来。预览没问题就一键下载 Excel，表头、合计行、颜色、格式全都帮你搞好了。

说几个细节：
- 支持多月数据累计对比（比如 1-3 月 vs 去年同期）
- 核心指标自动算：损益金额、WX、FIT 三列
- 输出的 Excel 是带样式的（深蓝色表头、合计行高亮），直接可以拿去汇报

**这是怎么做出来的？**

不是我从零设计了一个"月报分析平台"。是有一次月报催得急，我跟 AI 说：这是我的 CSV，这些是我要的维度，帮我写个脚本。跑通了之后，我发现每个月都要干这事，就逐步加了 Web 界面、Excel 导出、多月对比。

这就是工具箱思维 —— **先解决问题，再沉淀工具**。

### 案例二：Reverse Engineer — 迭代了 15 个版本（讲方法论）

这个例子特别适合说"方法论"。

有一次我需要核对一份内部截图报表的数字，跟原始 CSV 对不上。问题是：截图里的数字到底是千元还是万元？分类逻辑是什么？

听起来很简单对吧？但实际上 —— 我写了 **15 个版本**的脚本才搞定。

> reverse_engineer.py → v2 → v3 → ... → v9 → ks → ks2 → ks3 → ks4 → final → final2

从第一版"假设单位是千元"，到 v9 "确认是万元，千元÷10=万元"，到 final 版修正具体数值 —— 每一版都是在跑完上一版之后发现"哦，原来这里理解错了"。

**这里面的方法论是什么？**

不是说我蠢了 15 次。是这种数据对齐的工作，本质上就是一个"假设-验证-修正"的循环。以前我可能会在 Excel 里手动算半天，现在有 AI 帮我写脚本，每次修正假设只要改几行代码重新跑就行了。**迭代的成本变得极低**。

15 个版本听起来多，但每个版本可能就花了 5 分钟。总共一两个小时就把一个原本可能折腾几天的对齐工作搞定了。

## 📈 板块二：交易决策 — 从手搓到 AI 辅助（10 min）

### 核心方法论

这一块复杂度高很多 —— 回测引擎、数据管道、策略逻辑，动辄几千行代码。核心方法论是：**开发方式的进化**。

| 阶段 | 怎么干的 | 痛点 |
|------|---------|------|
| 📝 手搓代码 | 自己一行一行写 | 慢，出 bug 自己调 |
| 🌐 Claude 网页版 | 代码贴过去让它改 | 上下文丢失，贴来贴去 |
| 🚀 CodeBuddy | 整个项目级别地写 | 终于能理解整体架构 |

不是后面完全替代前面，而是**不同复杂度的任务适合不同的工具**。

### BMAD Quant System — 最能体现 coding 进化（深讲）

BMAD = Bloomberg Market Analysis & Data，449 个文件的完整量化系统：

- **Bloomberg 数据工具箱**：自动拉取 K 线/Tick 数据，UTC→北京时间
- **策略回测**：网格搜索最优参数，核心策略是周五夜盘
- **Web 管理界面**：数据探索器 + 策略管理
- **微信小程序**：随时随地看策略状态

这个系统手写要几个月。用 CodeBuddy 后它能理解整个项目结构，加新数据源时知道数据层、策略层、Web 层分别在哪。

**经验：** 复杂项目选 IDE 级 AI 工具；简单脚本网页版够了；但你自己要先想清楚架构。

### Strategy Lab — 策略回测实验室

Web 化的回测工具：选策略→设参数→一键回测→热力图出结果。

> 周五夜盘策略：周五 21:00 开仓，周六 02:00 平仓。流动性差的时候才有统计套利机会——回测验证过的。

## 📚 板块三：知识沉淀 — 让 AI 越用越懂你（5 min）

### 核心方法论

前面两个板块讲的是"用 AI 做事"，这个板块讲的是"让 AI 越来越懂你"。

大家用 ChatGPT 或者 Claude 肯定有个感受 —— 每次新对话都要从头说一遍背景。你上次跟它聊了一个小时的分析，下次开新对话它全忘了。

我的思路是：**给 AI 建一套知识库，让它有记忆**。

### 8 个技能模块

我给 Soros Agent（后面会讲）做了 8 个知识模块：

| 模块 | 教 AI 什么 |
|------|-----------|
| 央行政策分析 | 怎么看美联储、欧央行、日央行的政策 |
| 货币危机案例 | 1992 英镑狙击、1997 亚洲金融危机等经典案例 |
| 反射性理论 | 索罗斯的核心分析框架 |
| 宏观交易理论 | 利率平价、购买力平价等 |
| 经典交易战役 | 历史上著名的宏观交易案例 |
| 外汇市场分析 | 实时市场分析方法 |
| 腾讯大数据 | 内部数据平台查询指南 |
| 交易历史 | 经典交易案例库 |

每个模块都是一份结构化的 Markdown 知识文档。AI 在回答问题时会自动匹配相关的知识模块来增强回答质量。

### Memory 系统

更有意思的是记忆系统。AI 每次做完深度分析，会自动把有价值的内容归档：

| 记忆类型 | 数量 | 举例 |
|---------|------|------|
| **交易手册 (Playbook)** | 10 条 | 操作规则、进化协议 |
| **市场洞察 (Insights)** | 6 条 | "人民币是央行精心管理的笼中鸟——飞不高，也摔不重" |
| **外汇观点 (FX View)** | 2 条 | USD/CNH 2026年3月展望 |
| **历史教训** | 3 条 | 1992 英镑狙击的教训 |

### Reflection 引擎

还有一个自动反思机制：

- 每次深度分析后自动触发质量评估
- 四个维度打分：数据完整性 30% + 逻辑连贯性 30% + 证据质量 20% + 可操作性 20%
- 高质量分析（> 0.7 分）自动归档到知识库
- 低质量的会标记出哪里需要改进

**给大家的启发：** 不管你用什么 AI 工具，都可以想想怎么让它有"记忆"。哪怕只是把你常用的 prompt 和背景信息整理成一份文档，下次对话时贴进去，效果就会好很多。

## 🤖 板块四：Agent 探索 — 踩坑故事（8 min）

### 先说什么是 Agent

前面三个板块，AI 都是"被动的" —— 你给它一个任务，它做完了就完了。但 Agent 不一样，**Agent 可以自己规划步骤、自己调用工具、自己评估结果**。

简单类比：前面的工具像是一个很能干的实习生，你说干什么它就干什么。Agent 更像是一个有经验的分析师，你说"帮我看看最近美元走势"，它会自己决定先查汇率、再看新闻、再分析央行政策、再给你一个综合判断。

### Soros Agent — 索罗斯人设的 AI 外汇分析师

这是我花时间最多的一个项目。我给 AI 设定了一个"索罗斯"的人设 —— 用反射性理论（Reflexivity）来分析外汇市场。

**它能干什么？**

一个完整的分析流程（六步）：

1. **理解**你的问题
2. **检索记忆**——之前分析过什么、有什么相关的知识
3. **规划**——决定需要用哪些工具
4. **执行工具**——可以查汇率、看新闻、搜研报、查腾讯内部数据平台
5. **生成回答**——综合所有信息给出分析
6. **反思存储**——自动评估这次分析的质量，好的归档到知识库

它有 **19 个工具**（查汇率、查波动率、搜新闻、Perplexity 深度研究、保存洞察、查 WeData 平台等），会根据需要自己决定调用哪些，最多 5 轮工具调用。

**真实效果：一个有意思的洞察**

它分析过人民币汇率之后，自己总结出了这么一条：

> "人民币是一个被央行精心管理的笼中鸟——飞不高，也摔不重。"

还给出了概率判断：Base Case / Bull Case / Bear Case 各多少概率。这种输出我觉得是有参考价值的。

**翻车的地方**

当然也有翻车的时候：
- **Token 限制**：上下文一长就开始胡说八道，或者忘了前面说了什么
- **知识注入的取舍**：8 个技能模块全塞进去 token 爆了，不塞又不够用。最后用了一个简易 RAG，根据问题动态匹配相关知识
- **幻觉问题**：有时候会编造不存在的数据。所以核心原则是"知之为知之，不知为不知"，做了 Claim Verification 工具来验证关键论点

### 林徽因 Agent — 验证框架通用性

写完 Soros Agent 之后，我想验证一件事：**这套架构换个人设还能跑吗？**

于是做了一个林徽因 Agent —— 以中国第一位女建筑学家林徽因（1904-1955）的人设，跟你聊诗歌、建筑、美学。

结果换个 System Prompt、换个端口（8902），核心架构完全复用。这说明 Agent 的框架是可以通用的——不管你想做一个金融分析师、历史学家、还是产品经理 AI，底层都是那套 Agent Loop。

### 简略提：更远的方向

- **老查理 Multi-Agent**：在探索多个 AI Agent 组成"投委会"，每个 Agent 代表一个投资流派（价值派、成长派、量化派等），互相辩论后给出综合判断。还在设计阶段。
- **OpenClaw 框架**：想把 Agent 的开发范式标准化——Soros Agent 里已经有了雏形（Agent Loop + Skill + Memory），未来希望做成一个通用框架。

## 💡 收尾：经验总结 + 给大家的建议（4 min）

### 三条方法论回顾

| # | 方法论 | 一句话 |
|---|--------|-------|
| 1 | **工具箱思维** | 先解决眼前问题，再沉淀成通用工具。别一上来就想做平台。 |
| 2 | **开发方式进化** | 不同复杂度用不同工具。简单脚本用网页版 AI，复杂项目用 IDE 级 AI。 |
| 3 | **知识沉淀** | 让 AI 有记忆。哪怕只是整理常用 prompt，也比每次从零开始强。 |

### 给非技术背景同事的建议

你不需要会写代码才能用好 AI。几个可以立即上手的方向：

1. **重复性的文档/报表工作** → 试试用 AI 写模板、做汇总
2. **信息检索和分析** → 把背景信息喂给 AI，让它帮你总结
3. **知识管理** → 把你的工作方法、常见问题整理成文档，作为 AI 的"参考手册"
4. **CodeBuddy** → 如果你想尝试做简单的工具，CodeBuddy 可以让你用自然语言描述需求，它帮你写代码

### 轻松收尾

今天分享的这些工具，都不是什么了不起的大项目。每一个都是从一个很具体的工作痛点开始的。

AI 最大的价值不是替代你，而是**让你把时间花在真正需要判断力的事情上**。

谢谢大家！
